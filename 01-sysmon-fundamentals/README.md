# Background Theory

## What is Sysmon?

Sysmon (System Monitor) is a Microsoft Sysinternals utility that provides detailed logging of system activity. It records events such as process creation, network connections, DNS queries, file creation, and registry modifications, giving security analysts greater visibility into endpoint behavior.

---

## Why SOC Analysts Use Sysmon

Sysmon extends the default Windows logging capabilities by collecting additional telemetry that helps analysts:

- Investigate suspicious activity.
- Detect malicious processes.
- Monitor network connections.
- Track file creation.
- Observe registry modifications.
- Support incident response.

---

## Windows Event Logs vs Sysmon

| Windows Event Logs | Sysmon |
|--------------------|---------|
| Built into Windows | Requires installation |
| Limited endpoint visibility | Detailed endpoint telemetry |
| Focuses on system events | Focuses on security-relevant activity |
| Basic auditing | Advanced monitoring |

---

## Sysmon Log Location

Sysmon events are stored in:

```text
Applications and Services Logs
└── Microsoft
    └── Windows
        └── Sysmon
            └── Operational
```

# Installation

Installed Sysmon using PowerShell with administrative privileges.

Command used:

```powershell
.\Sysmon64.exe -i
```

Verified the Sysmon service using:

```powershell
Get-Service Sysmon64
```
